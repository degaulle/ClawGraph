Sample of authored files (with descriptions):
  - codex-rs/app-server/src/codex_message_processor.rs: # CodexMessageProcessor

This file handles all incoming client requests for Codex's AI assistant features. It processes JSON-RPC messages that control thread (conversation) lifecycle, user interactions, authentication, file searching, and configuration management.

The processor acts as a central hub that translates client API calls into operations on active threads, manages login sessions, handles file discovery, lists available AI models and collaboration modes, and orchestrates event delivery back to clients. It supports both legacy "conversation" APIs and newer "thread/turn" APIs, manages user authentication (API key and ChatGPT), and coordinates real-time event streaming to connected clients listening to specific threads.
  - codex-rs/core/src/thread_manager.rs: This file manages the lifecycle of conversation threads in Codex. It creates new threads, loads previous threads from saved history, allows users to fork threads at specific points in a conversation, and keeps track of all active threads in memory. It also handles setup tasks like monitoring for skill updates and managing authentication and AI model availability for each thread.
  - codex-rs/feedback/src/lib.rs: This file implements a feedback collection and reporting system that captures detailed diagnostic information (logs and metadata) from the Codex application and uploads it to Sentry for debugging and issue tracking. It maintains a rolling buffer of recent logs so that when users report problems, the system can attach comprehensive context about what happened, then sends that data along with user-provided classification (like "bug" or "bad result") to a centralized error monitoring service.
  - codex-rs/protocol/src/protocol.rs: # Summary

This file defines the communication protocol between Codex (an AI coding agent) and its clients. It establishes the message format for two-way streaming communication: clients send **submissions** (user requests, approvals, configuration changes) and receive **events** (agent responses, command output, status updates). The file also specifies execution policies for sandboxing shell commands, approving actions, and managing file access permissions—allowing clients to control how much freedom the agent has when running code on the user's system.
  - codex-rs/tui/src/bottom_pane/feedback_view.rs: This file implements a feedback submission dialog that lets users report bugs, comment on code quality, or provide general feedback about Codex. When submitted, it captures the user's optional note and uploads it along with session logs (if the user consents), then displays a confirmation message with links to follow up on GitHub or internal channels depending on whether they're an employee.
  - codex-rs/login/src/device_code_auth.rs: This file implements a device-code login flow that allows users to authenticate with their ChatGPT account through a browser-based code exchange. When a user runs the login command, the system generates a unique code and verification URL, displays them in the terminal with clear instructions, and then polls the server in the background until the user completes authentication in their browser—at which point login credentials are securely saved locally.
  - codex-rs/tui/src/onboarding/auth.rs: This file handles the onboarding authentication flow for Codex, letting users sign in via three methods: ChatGPT account, device code, or API key. It manages the visual screens users see during login (prompts, input fields, success messages) and processes user interactions like keyboard input and API key pasting to guide them through whichever authentication method they choose.
  - codex-rs/cli/src/login.rs: This file handles user authentication for the Codex command-line tool, supporting multiple login methods: ChatGPT browser-based login, API key login, and device code login. It provides commands to log in, check login status, and log out, while respecting configuration rules that may restrict which login methods are available.
  - codex-rs/cli/src/main.rs: This file is the main entry point for the Codex command-line tool. It parses user commands and routes them to the appropriate features—whether that's starting an interactive chat session, running code non-interactively, managing login credentials, resuming past conversations, or accessing experimental tools like the desktop app and cloud task browser. It also handles configuration overrides, feature flags, and platform-specific sandbox tools.
  - codex-rs/core/src/env.rs: This file detects the computing environment where Codex is running—checking whether it's in Windows Subsystem for Linux, running over SSH, in a CI/CD system, or without a graphical display. The product uses these detections to adapt its behavior, such as avoiding browser-based login flows when there's no GUI available.
