  - codex-rs/app-server/src/codex_message_processor.rs: # CodexMessageProcessor

This file handles all incoming client requests for Codex's AI assistant features. It processes JSON-RPC messages that control thread (conversation) lifecycle, user interactions, authentication, file searching, and configuration management.

The processor acts as a central hub that translates client API calls into operations on active threads, manages login sessions, handles file discovery, lists available AI models and collaboration modes, and orchestrates event delivery back to clients. It supports both legacy "conversation" APIs and newer "thread/turn" APIs, manages user authentication (API key and ChatGPT), and coordinates real-time event streaming to connected clients listening to specific threads.
  - codex-rs/cli/src/debug_sandbox.rs: This file is the command-line interface for testing Codex in different security sandbox environments (macOS Seatbelt, Linux Landlock, and Windows sandboxes). It loads configuration, sets up the appropriate sandbox, runs a user-supplied command inside it, and optionally logs any security policy violations that occurred.
  - codex-rs/core/src/exec_env.rs: This file controls which environment variables (like PATH, API keys, etc.) are passed to code that Codex executes, based on a configurable security policy. It filters out sensitive variables by default (those containing "KEY", "SECRET", or "TOKEN"), allows customization through allowlists and blocklists, and automatically injects a thread identifier so executed code can identify which Codex session it belongs to.
  - codex-rs/core/src/tasks/user_shell.rs: This file handles the execution of shell commands that users run directly within the Codex environment. It takes a command string, runs it in the user's default shell (supporting features like pipes and redirects), captures the output, and reports back the results—including success/failure status, exit codes, and stdout/stderr—to the session so the user sees what happened.
  - codex-rs/core/src/tools/handlers/shell.rs: This file handles the execution of shell commands when Codex needs to run terminal operations. It receives command requests from the AI model, validates them for safety and permission requirements, and then executes them in a sandboxed environment while tracking the results. It also prevents certain dangerous operations from running if security policies don't permit them.
  - codex-rs/core/src/unified_exec/process_manager.rs: This file manages the lifecycle of running shell commands and processes in Codex. It handles starting new command sessions, collecting their output within time limits, routing input back to running processes, and cleaning up old sessions when resource limits are reached. It also enforces environment defaults (like disabling colored output) and implements a strategy to prune the oldest or already-exited processes when too many accumulate.
  - codex-rs/linux-sandbox/tests/suite/landlock.rs: This file contains automated security tests that verify the Linux sandbox properly restricts what code can do—such as blocking network access, preventing writes to system directories, and protecting sensitive configuration files from being overwritten. It ensures that when Codex executes user code in a sandboxed environment, malicious or accidental operations are safely blocked.
