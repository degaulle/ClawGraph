  - codex-rs/tui/src/bottom_pane/chat_composer.rs: # ChatComposer Summary

The `ChatComposer` is the interactive text input widget for the chat interface. It manages everything the user types, pastes, and attaches before sending a message.

## Core Responsibilities

**Text Editing & Input**
- Handles typing, pasting, and deletion with a rich textarea that supports atomic elements (like image placeholders and slash commands)
- Detects "paste bursts" (rapid input that looks like pasted content from the clipboard) to distinguish them from human typing
- Expands large pastes into placeholder elements to keep the UI responsive, then restores full content on submission

**Attachments**
- Manages local image files attached by the user (with placeholder labels like `[Image #1]`)
- Tracks remote image URLs separately and renders them as non-editable rows above the textarea
- Automatically renumbers local image placeholders when remote images are added or removed

**History Navigation**
- Stores submitted messages in local in-session history and (via async lookup) persistent cross-session history
- Supports shell-style Up/Down arrow navigation to recall previous messages, with cursor positioned at end-of-line for repeated navigation

**Slash Commands & Popups**
- Recognizes `/command` syntax and shows an autocomplete popup
- Allows Tab to complete commands, Enter to dispatch them
- Supports custom prompts with variable expansion (`$USER`, `$1`–`$9`, etc.)
- Also manages `@file` search popups and `$mention` popups for skills and connectors

**Submission Modes**
- "Steer mode": Enter submits immediately, Tab queues while a task runs
- Standard mode: Enter queues while a task runs, otherwise submits
- Validates commands before submission and reports errors

**State Preservation**
- Tracks pending large-paste payloads so they survive editing and are expanded only on submit
- Records mention bindings so `$skill` references resolve correctly even after history recall
- Clears text on Ctrl+C while preserving it in history for undo-like recall

## Key User-Facing Behavior

- **Paste detection**: Small pastes insert directly; large pastes use placeholders to avoid lag
- **IME-friendly**: Non-ASCII input (from input methods) is never held, avoiding dropped character feeling
- **Image attachment**: Users can paste image file paths and they're automatically detected and attached
- **Shell-like history**: Repeated Up/Down keeps navigating backwards/forwards without stopping at interior cursor positions
- **Read-only mode**: Can be temporarily disabled (e.g. during setup) with a placeholder message shown instead
