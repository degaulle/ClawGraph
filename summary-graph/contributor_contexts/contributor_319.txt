  - codex-rs/core/src/codex.rs: # Summary

This file implements the **core orchestration engine for the Codex AI assistant**. It manages conversation sessions, coordinates model interactions, handles user input, processes tool calls, and maintains conversation history.

## Key Responsibilities

**Session Management**: The `Codex` struct serves as the high-level public interface—clients submit operations through it and receive events back. The `Session` struct manages the actual conversation state, including history, configuration, active turns, and communication with backend services (models, tools, authentication).

**Turn Processing**: The `run_turn` function executes a single conversation turn by repeatedly calling the model, collecting its responses, executing requested tools, and feeding results back to the model until it produces a final message rather than requesting more tool calls.

**Tool Integration**: Builds and routes tool calls to appropriate handlers (shell commands, file operations, web search, MCP servers, etc.), managing approval workflows and sandbox policies along the way.

**History & Context Management**: Tracks conversation history, applies automatic compaction when reaching token limits, handles session resumption and forking, and manages initial context seeding with permissions and environment information.

**Configuration & Personalization**: Applies per-turn configuration overrides (model, reasoning effort, approvals, sandbox policies, personality preferences), manages MCP server connections, and handles feature flags.

**Event Streaming**: Provides a real-time event channel to clients showing model output deltas, plan segments, token usage, and system notifications as they happen.
  - codex-rs/core/src/compact.rs: This file handles **conversation compaction** — the process of summarizing long chat histories to fit within the model's memory limits so Codex can continue working effectively on extended conversations.

When a conversation gets too long, this code automatically summarizes older messages (asking the AI model to create a condensed version), then replaces the lengthy history with that summary plus recent user messages. It manages retries if the compaction request fails, handles edge cases like oversized messages, and ensures important context (like project instructions and environment details) stays fresh and isn't lost in the compression process.
  - codex-rs/core/src/context_manager/history.rs: This file maintains and manages the conversation history for Codex—the back-and-forth exchange of messages, tool calls, and responses between the user and the AI model. It tracks what's been said, estimates how many tokens (units of text) have been used, truncates long outputs to fit within limits, and allows the product to rewind the conversation by removing recent user turns or cleaning up outdated entries.
  - codex-rs/core/src/event_mapping.rs: This file translates conversation data from the model's internal format into the structured format used throughout the Codex product. It takes raw message responses and converts them into standardized turn items that represent user messages, assistant replies, reasoning steps, and web search actions—filtering out internal metadata like instructions and image markup tags along the way.
  - codex-rs/core/src/lib.rs: This file is the main entry point for Codex Core, the foundational library that powers the Codex AI assistant. It organizes and exposes all the internal components the product needs—such as authentication, model management, sandboxing, conversation history, shell command execution, security policies, and AI model connections—so that other parts of the system can use them consistently. Think of it as the "blueprint" that tells the rest of the product what building blocks are available and how to access them.
  - codex-rs/core/src/session_prefix.rs: This file defines how the system recognizes special configuration messages that provide context to the AI model across multiple conversation turns—such as environment setup or information about failed attempts—without treating them as actual user inputs that would create new conversation turns.
  - codex-rs/core/src/tasks/mod.rs: This file manages the execution of different types of AI conversation tasks (like regular chat, code reviews, and undo operations) as background work within a session. It provides a framework for launching, tracking, and canceling these tasks, handling cleanup when they complete or get interrupted by the user, and ensuring the session stays synchronized with task progress.
  - codex-rs/core/tests/suite/abort_tasks.rs: This file tests the interrupt feature that allows users to stop long-running tasks in Codex. It verifies three key behaviors: that interrupting a task properly stops it and signals completion, that the system remembers what was interrupted and reports it to the AI model on the next turn, and that a marker is added to the conversation history so the model knows a turn was aborted.
  - codex-rs/core/src/shell_snapshot.rs: This file captures and stores snapshots of the user's shell environment (functions, aliases, variables, settings) so that Codex can recreate that environment when needed for code generation or execution tasks. It handles creating shell-specific snapshot files, validating them, and cleaning up old snapshots to manage disk space.
