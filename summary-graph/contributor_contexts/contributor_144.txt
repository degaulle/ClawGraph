  - codex-rs/tui/src/bottom_pane/chat_composer.rs: # ChatComposer Summary

The `ChatComposer` is the interactive text input widget for the chat interface. It manages everything the user types, pastes, and attaches before sending a message.

## Core Responsibilities

**Text Editing & Input**
- Handles typing, pasting, and deletion with a rich textarea that supports atomic elements (like image placeholders and slash commands)
- Detects "paste bursts" (rapid input that looks like pasted content from the clipboard) to distinguish them from human typing
- Expands large pastes into placeholder elements to keep the UI responsive, then restores full content on submission

**Attachments**
- Manages local image files attached by the user (with placeholder labels like `[Image #1]`)
- Tracks remote image URLs separately and renders them as non-editable rows above the textarea
- Automatically renumbers local image placeholders when remote images are added or removed

**History Navigation**
- Stores submitted messages in local in-session history and (via async lookup) persistent cross-session history
- Supports shell-style Up/Down arrow navigation to recall previous messages, with cursor positioned at end-of-line for repeated navigation

**Slash Commands & Popups**
- Recognizes `/command` syntax and shows an autocomplete popup
- Allows Tab to complete commands, Enter to dispatch them
- Supports custom prompts with variable expansion (`$USER`, `$1`â€“`$9`, etc.)
- Also manages `@file` search popups and `$mention` popups for skills and connectors

**Submission Modes**
- "Steer mode": Enter submits immediately, Tab queues while a task runs
- Standard mode: Enter queues while a task runs, otherwise submits
- Validates commands before submission and reports errors

**State Preservation**
- Tracks pending large-paste payloads so they survive editing and are expanded only on submit
- Records mention bindings so `$skill` references resolve correctly even after history recall
- Clears text on Ctrl+C while preserving it in history for undo-like recall

## Key User-Facing Behavior

- **Paste detection**: Small pastes insert directly; large pastes use placeholders to avoid lag
- **IME-friendly**: Non-ASCII input (from input methods) is never held, avoiding dropped character feeling
- **Image attachment**: Users can paste image file paths and they're automatically detected and attached
- **Shell-like history**: Repeated Up/Down keeps navigating backwards/forwards without stopping at interior cursor positions
- **Read-only mode**: Can be temporarily disabled (e.g. during setup) with a placeholder message shown instead

This file contains comprehensive integration tests for the **ChatWidget**â€”the central UI component that adapts incoming protocol events (from the Codex agent backend) into a terminal UI display and handles user input.

## Purpose

The tests verify that ChatWidget correctly:
- **Handles agent events** (messages, command execution, approvals, reasoning, plans, etc.) and renders them in the chat history
- **Processes user input** (text submission, keyboard shortcuts, image attachments, slash commands) and translates it into operations sent back to the backend
- **Manages UI state** (task running/idle, modal popups, composer text, status indicators, rate limit warnings, etc.)
- **Preserves user data** (text elements, local/remote images, mention bindings, queued messages) through lifecycle events like interrupts and reconnects
- **Renders correctly** at various terminal sizes and UI configurations using snapshot tests

## Key Test Categories

**Message & History Flow**: Tests for replayed messages, multi-turn conversations, image attachments, text elements, and proper history cell rendering.

**Execution & Approval**: Tests for command execution display, approval modals (exec/patch), decision history, unified exec processes, and background terminal management.

**User Input & Submission**: Tests for message submission with images/elements, keyboard shortcuts (Enter/Shift+Enter), Alt+Up history recall, Ctrl+C clearing, and Tab queuing.

**Status & Indicators**: Tests for working status display, context window percentage, token counting, stream reconnection, and runtime metrics.

**Rate Limiting & Prompts**: Tests for rate limit warnings, model switch suggestions, and plan implementation popups.

**Collaboration Modes & Features**: Tests for mode switching (/plan, /collab), personality selection, reasoning effort, and feature gates.

**Review Mode**: Tests for review request handling, commit picker, custom prompts, and branch selection.

**Modals & Popups**: Tests for approval selection, model/reasoning pickers, experimental features, feedback submission, and app connectors.

**Edge Cases**: Tests for interrupted turns, small terminal heights, disabled commands, constrained configs, and invalid status line items.

The tests use snapshot assertions to detect unintended layout or message changes; many render to a VT100 terminal backend to capture the exact visual output.
