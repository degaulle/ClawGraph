  - codex-rs/app-server/src/codex_message_processor.rs: # CodexMessageProcessor

This file handles all incoming client requests for Codex's AI assistant features. It processes JSON-RPC messages that control thread (conversation) lifecycle, user interactions, authentication, file searching, and configuration management.

The processor acts as a central hub that translates client API calls into operations on active threads, manages login sessions, handles file discovery, lists available AI models and collaboration modes, and orchestrates event delivery back to clients. It supports both legacy "conversation" APIs and newer "thread/turn" APIs, manages user authentication (API key and ChatGPT), and coordinates real-time event streaming to connected clients listening to specific threads.
  - codex-rs/app-server/tests/common/auth_fixtures.rs: This file provides testing utilities for creating fake ChatGPT authentication credentials. It allows test engineers to easily generate mock login data (tokens, user IDs, email addresses, etc.) and write them to a test environment, so they can verify that the product correctly handles user authentication without needing real OpenAI accounts.
  - codex-rs/chatgpt/src/chatgpt_token.rs: This file manages the ChatGPT authentication token used by the application. It provides functions to store and retrieve the user's token in memory, and includes logic to load the token from the system's authentication configuration file when the application starts up.
  - codex-rs/cli/src/login.rs: This file handles user authentication for the Codex command-line tool, supporting multiple login methods: ChatGPT browser-based login, API key login, and device code login. It provides commands to log in, check login status, and log out, while respecting configuration rules that may restrict which login methods are available.
  - codex-rs/core/src/auth.rs: This file manages user authentication for the Codex product, handling login, logout, and token refresh across three auth methods: API keys, ChatGPT accounts with stored credentials, and externally-managed ChatGPT tokens. It loads user credentials from storage (or environment variables), keeps them in memory, automatically refreshes expired tokens when needed, and enforces login restrictions like requiring a specific authentication method or workspace. The file also provides recovery logic when API requests fail with authentication errors—it can reload fresh credentials from disk or request new tokens from an external service before retrying.
  - codex-rs/core/src/auth/storage.rs: This file handles how Codex stores and retrieves user authentication credentials (like API keys and tokens). It supports four different storage methods: saving to a local file, saving to the system's secure credential storage (keyring), automatically choosing between the two, or keeping credentials in memory only for the current session. The file includes the logic to read from, write to, and delete credentials using whichever storage method the user has configured.
  - codex-rs/core/tests/suite/client.rs: This file is a comprehensive test suite that verifies how the Codex client sends requests to AI model APIs and processes responses. It tests critical product features like authentication, conversation history management, user instructions, model configuration options (reasoning effort, verbosity), rate limiting, error handling, and resuming saved conversations—essentially ensuring that user inputs are properly formatted and transmitted to the backend with all necessary context and settings.
  - codex-rs/login/src/lib.rs: This file serves as the main entry point for user login and authentication in Codex, exposing the key features needed to log in users (like device code authentication and a local login server) along with core authentication utilities for managing API keys, tokens, and user sessions.
  - codex-rs/login/src/server.rs: # LoginServer

This file runs a lightweight web server that handles the login flow for Codex. When a user initiates login, it starts a local server, directs them to OpenAI's authentication page, waits for them to complete the login, captures the returned credentials, and saves them locally for future use. It also handles graceful shutdown and retry logic if the server port is already in use from a previous login attempt.
  - codex-rs/login/tests/suite/device_code_login.rs: This file tests the device code login flow—a method that lets users authenticate by entering a code on a website rather than typing credentials directly into the app. It simulates the entire login process by mocking server responses and verifies that credentials are properly saved, workspace restrictions are enforced, and errors are handled gracefully.
