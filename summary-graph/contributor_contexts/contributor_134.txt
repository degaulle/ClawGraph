When a user initiates login, it starts a local server, directs them to OpenAI's authentication page, waits for them to complete the login, captures the returned credentials, and saves them locally for future use.   - codex-rs/app-server/src/codex_message_processor.rs: # CodexMessageProcessor

This file handles all incoming client requests for Codex's AI assistant features. It processes JSON-RPC messages that control thread (conversation) lifecycle, user interactions, authentication, file searching, and configuration management.

The processor acts as a central hub that translates client API calls into operations on active threads, manages login sessions, handles file discovery, lists available AI models and collaboration modes, and orchestrates event delivery back to clients. It supports both legacy "conversation" APIs and newer "thread/turn" APIs, manages user authentication (API key and ChatGPT), and coordinates real-time event streaming to connected clients listening to specific threads.
  - codex-rs/cli/src/main.rs: This file is the main entry point for the Codex command-line tool. It parses user commands and routes them to the appropriate features—whether that's starting an interactive chat session, running code non-interactively, managing login credentials, resuming past conversations, or accessing experimental tools like the desktop app and cloud task browser. It also handles configuration overrides, feature flags, and platform-specific sandbox tools.
  - codex-rs/mcp-server/src/lib.rs: This file is the entry point for an MCP (Model Context Protocol) server that acts as a bridge between a user's command line and the Codex system. It sets up three coordinated tasks—reading requests from the user's input, processing those requests through the Codex engine, and sending responses back to the user—while managing configuration settings and logging along the way.
  - codex-rs/mcp-server/src/main.rs: This file is the entry point that starts the MCP (Model Context Protocol) server, which enables Codex to communicate with external tools and integrations. It initializes the server with default configuration settings and prepares the system to handle requests from connected clients.
  - codex-rs/mcp-server/src/message_processor.rs: This file is the central message router for the Codex MCP server—it receives requests and notifications from external clients, decides what to do with them, and orchestrates the appropriate responses. It handles initialization, processes tool calls that start Codex sessions or continue existing conversations, manages cancellations, and routes unsupported requests back with error messages.
  - codex-rs/mcp-server/src/outgoing_message.rs: This file handles sending messages from the Codex server back to clients. It provides a message sender that queues outgoing requests, responses, notifications, and errors, while also managing callbacks so the server can receive replies to requests it sends to clients.
  - codex-rs/core/src/client.rs: This file manages the communication layer between Codex and AI model providers' APIs. It handles session-level configuration (like authentication and provider selection) that persists across multiple conversation turns, while accepting turn-specific settings (like model choice and reasoning effort) as explicit parameters to keep those lifetimes clear. The file implements two main abstractions: a session-scoped `ModelClient` that holds stable connection setup, and a turn-scoped `ModelClientSession` that manages per-turn state like WebSocket connections and sticky-routing tokens. It supports both HTTP and WebSocket transports for streaming responses, with automatic fallback from WebSocket to HTTP when needed, and handles features like request compression, auth token refresh, and incremental request appending to avoid resending data.
